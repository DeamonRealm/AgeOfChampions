Contribution

Main Menu:

Initial screen with few buttons that show the options available to the player, options such as a Tutorial, Wiki (develop team Github), Exit, Report Bug and Single Player. Single Player Button will display 3 more buttons (Standard Game, Load Saved Game and Load Alpha Game). Tutorial and Wiki Links made in collaboration with Ferran Martí


Player:

In-Game Hud: Base Module for Player information Display and Entities Managment. It coordinates all game options spliting their functionality into 4 submodules: Game Panel, Action Panel, Selection Panel and Minimap


Game Panel:

Diplayed in the Top of the screen as a information bar, in the top left corner it shows own resources and population available, on the top right corner there's the Pause Menu Button with game options (quit game, save, load and cancel)


Action Panel:

Manage Units and Buildings actions such as spawn units, research tecnologies, use skills and skill tree (champions), build buildings (villagers), upgrade civilization or even kill selected unit. Every action available have text information displayed when overing the button. It interacts with Game Panel to manage resources and Selection Panel to Update Units state.


Selection Panel:

A family of basic isometric primitives used in logic to get the entities that are located in a certain zone. Primitives are used in heroes abilities and units collisions and interaction areas.


Minimap:

Submodule that prints basic map information, blue marks are own units/buildings and red are enemies ones, fog of war updates acording to map state. Player can interact with it and move the camera or even move units to one point or another. When under attack an alert apear and point the battle position, the current Camera position is also shown with a rect in the minimap.


Player Save and Load:

Save current match information, also Load and update submodules if player chose to load match.


Entities Information Managment:

Player and AI storage of information related to units, buildings, tecnologies and skills pluss system to use it. Player and AI Data containers are different but they may contain: unit/building/resource types, base types which is the origin of the entity (militia base type from Longswordman), produced from (needed to porduce unit/building/tecnology), tecnology type that will update it, age and lvl where they are available, resources and population cost and short description of element shown as help for the player.


Entities Life Bar:

If one entity is selected it shows it's own life adobe him, if a group of units are selected all their life bars are shown and updated.


Civilization Upgrades:

Civilization Research system, manage research and every time tecnology is researched it upgrade entities in need. This might change civilizations age and it's consequent change of buildings stats, sprites and actions available. Some research may affect units or buffs.


Units Upgrades:

Doing Research, units may be improved to next level, one militia evolve to man-at-arms and so on. This type of update need to iterate all current units and units inside production queue and update them to their next level. If units that evolve are Allies Action Panels must be updated too.


AI Civilization Upgrade:

AI improve their tecnology and does research as if it was a player, i did the Ai research system that evolve current AI within it's possibilities and manage the resources avaiable to develop them. Also if next tecnology to research need a production type (Stable) and there's no stable builded it search for a villager to build it and then start the research.


AI Buildings Managment:

A module that manages all the buffs logic and particles. Buffs definitions are loaded and entities can use them to apply buffs. Buff definitions have the same organization as module animation so all definitions are organized in a tree. This module was elaborated in contribution with Sebas.


AI Units Production Managment:

A basic module that handle input and let the user configure the input keys using the config xml. Only keyboard inputs are programmable.


AI Save and Load:

This module has the functionality to reproduce ovg video format and uses theoraplay library to do it. This functionality is only used in the initial video where the team logo is showed.


Win & Lose Contitions:

Only basic modules like renderer, file system, GUI, … Are loaded at the app start all the other modules related with the game content like animation, entities manager, buff manager, … Are loaded in a load screen when the party starts. To maintain a frame rate in the load screen a load time per frame is specified and when that time is wasted the current module that where loading save the current load point and let the app do an iteration to update the other active modules. To see the frame rate there’s a loading particle in the right bottom corner.
